**As a** content distributor, **I want to** POST a single prompt and target platforms and receive persisted, cached, platform-specific outputs, **so that** I generate all my distribution content in one call and never pay for the LLM twice on the same request.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (POST /generate) | Untrusted JSON body crosses here; validated by global ValidationPipe + GenerateRequestDto |
| API → LLM provider | Prompt forwarded to OpenAI via LLMProvider; output schema-constrained by MusicConceptSchema |
| API → PostgreSQL | PersistenceService writes request + result rows via Prisma parameterized queries |
| API → Redis (cache) | Deterministic sha256 key; stored response includes a real requestId only after persistence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | POST /generate body | mitigate | GenerateRequestDto @IsIn rejects unknown platforms; @MaxLength(500) limits prompt blast radius; global ValidationPipe whitelist:true strips unknown fields (D-18) |
| T-02-05 | Tampering | cache key collision | mitigate | sha256 over `prompt + '|' + sortedPlatforms`; '|' separator cannot appear in platform names so namespaces cannot collide (D-08) |
| T-02-06 | Denial of Service | LLM cost amplification | mitigate | Phase 1 rate limit (3/min/IP) global; cache returns identical requests without an LLM call (CACHE-02) |
| T-02-07 | Information Disclosure | DB write failure | mitigate | persist wrapped in try/catch; error logged internally, response uses crypto.randomUUID() fallback requestId — no stack trace to client (D-15) |
| T-02-08 | Denial of Service | one processor throws | mitigate | Promise.allSettled + buildFallback ensures partial failure never fails the whole request (PIPE-04, D-12) |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new packages (RESEARCH Package Legitimacy Audit) — no install task, no slopcheck required |
</threat_model>
