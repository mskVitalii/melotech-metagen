<objective>
Implement the LLM integration layer on top of the walking skeleton: the `LLMProvider` abstract class (a runtime-surviving NestJS DI token), the `OpenAIProvider` that calls OpenAI's structured-output API via `chat.completions.parse()` + `zodResponseFormat()`, the mandatory refusal guard (400 not 500), timeout/retry config, and the shared `MusicConceptSchema`. This is the vertical slice that proves the system can turn a prompt into a schema-conforming `MusicConcept`.

Purpose: Phase 2's GenerationService and platform processors inject `LLMProvider` (never the OpenAI SDK directly) and consume `MusicConcept`. Establishing the abstraction and the shared schema here keeps vendor SDK calls out of all downstream code.

Output: A working, unit-tested LLM module exporting `LLMProvider`, plus the canonical `MusicConceptSchema`.

Implements PIPE-02. Honors D-04 (abstract class as DI token), D-05 (parse() + zodResponseFormat, never json_object), D-06 (model pinned via OPENAI_MODEL), D-07 (refusal guard -> 400), D-08 (timeout 30000, maxRetries 2), D-09 (MusicConcept schema). ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User prompt → OpenAI API | Untrusted prompt text crosses into the LLM call |
| Environment → OpenAIProvider | OPENAI_API_KEY injected via ConfigService; never logged |
| OpenAI response → application | Model output crosses back; must be schema-validated and refusal-checked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | OPENAI_API_KEY | mitigate | Read only via ConfigService from env; never logged or returned in responses; Zod EnvSchema ensures non-empty at boot (ASVS V14) |
| T-01-06 | Tampering | Prompt injection via generateStructured | mitigate | zodResponseFormat constrains output to MusicConceptSchema server-side; no dynamic SQL/command built from prompt or model output (ASVS V5) |
| T-01-07 | Denial of Service | Hanging OpenAI calls | mitigate | OpenAI client `timeout: 30_000`, `maxRetries: 2` (D-08) bound request duration |
| T-01-08 | Error Handling / Info Disclosure | Refusal path | mitigate | message.refusal/parsed===null -> BadRequestException (400) with safe message; no stack trace leaked; not a 500 (D-07, ASVS V7) |
| T-01-SC | Tampering | openai/zod npm installs | mitigate | `openai`, `zod` are [Approved] in RESEARCH Package Legitimacy Audit; no [ASSUMED]/[SUS] packages -> no checkpoint required |
